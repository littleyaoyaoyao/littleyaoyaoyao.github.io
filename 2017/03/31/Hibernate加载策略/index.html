标签 ： Hibernate 加载策略

加载策略

延迟加载：等到使用的时候才会加载数据
立即加载：不管使用不使用，都会立刻将数据加载

策略的应用

类级别的加载策略
关联级别的加载策略

类级别的加载策略

  1. get/load
  • get：立即查询数据库，将数据初始化
  • load：hbm文件中，class元素的lazy属性绝对类级别load方法的加载策略
    true : 先返回一个代理对象。使用代理对象的属性时，采取查询数据库
false : 与get一致，会理解加载数据

关联级别加载策略

在查询有关联关系的数据时，加载一方的数据是否需要将另一方理解查询出。

默认：关联的数据在使用时才会加载

一对多

  1. 使用lazy——-对于set

    <set name="orders" lazy="true">

  • true（默认值） : 是否对set数据使用懒加载
  • false : 集合将会被立即加载
  1. 使用fetch —— 对于set
    决定加载集合使用的sql语句
  • select(默认值) : 普通select查询
  • join : 表链接语句查询集合数据
  • subselect : 使用子查询加载集合数据,一次加载多个Customer订单
  1. 共六种情况排列组合
  • select + true : 默认值，会在使用集合时加载，普通slelect语句
  • select + false : 立刻使用selelct语句加载集合数据
  • join + true : 查询集合时使用表链接查询，会立刻加载集合数据
  • join + false : 查询集合时使用表链接查询，会立刻加载集合数据
  • subselect + true : 会在使用集合时记在，子查询语句
  • subselect + false : 会在查询用户时，立即使用子查询加载客户的订单数据
  1. extra 极其懒惰
    在使用集合的时候，没有使用集合的数据而是使用了集合的长度size，那么Hibernate会发送count查询数量而不会加载集合内数据。
多对一

此时应该配置在Order一方many-to-one中

  1. lazy
  • flase : 加载订单时，会立刻加载客户
  • proxy : 看客户对象的类加载策略来决定
  • no-proxy : 不做研究
  1. fetch
  • select : (默认值)使用普通select加载
  • join : 使用表链接加载数据
  1. 排列组合
  • select + proxy : 类加载策略为:lazy=false 同上
  • select + false : 加载订单时,立即加载客户数据.普通select语句加载客户.
  • join + proxy : 使用表链接查询订单以及对应客户信息.lazy属性无效
  • join + false : 使用表链接查询订单以及对应客户信息.lazy属性无效

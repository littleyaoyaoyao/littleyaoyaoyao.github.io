加载策略

延迟加载：等到使用的时候才会加载数据
立即加载：不管使用不使用，都会立刻将数据加载

策略的应用

类级别的加载策略

  1. get/load
  • get：立即查询数据库，将数据初始化
  • load：hbm文件中，class元素的lazy属性绝对类级别load方法的加载策略
    true : 先返回一个代理对象。使用代理对象的属性时，采取查询数据库
false : 与get一致，会理解加载数据

2.懒加载

关联级别加载策略

在查询有关联关系的数据时，加载一方的数据是否需要将另一方理解查询出。

默认：关联的数据在使用时才会加载

一对多

  1. 使用lazy——-对于set

    <set name="orders" lazy="true">

  • true（默认值） : 是否对set数据使用懒加载
  • false : 集合将会被立即加载
  1. 使用fetch —— 对于set
    决定加载集合使用的sql语句
  • select(默认值) : 普通select查询
  • join : 表链接语句查询集合数据
  • subselect : 使用子查询加载集合数据,一次加载多个Customer订单
  1. 共六种情况排列组合
  • select + true : 默认值，会在使用集合时加载，普通slelect语句
  • select + false : 立刻使用selelct语句加载集合数据
  • join + true : 查询集合时使用表链接查询，会立刻加载集合数据
  • join + false : 查询集合时使用表链接查询，会立刻加载集合数据
  • subselect + true : 会在使用集合时记在，子查询语句
  • subselect + false : 会在查询用户时，立即使用子查询加载客户的订单数据
  1. extra 极其懒惰
    在使用集合的时候，没有使用集合的数据而是使用了集合的长度size，那么Hibernate会发送count查询数量而不会加载集合内数据。
多对一

此时应该配置在Order一方many-to-one中

  1. lazy
  • flase : 加载订单时，会立刻加载客户
  • proxy : 看客户对象的类加载策略来决定
  • no-proxy : 不做研究
  1. fetch
  • select : (默认值)使用普通select加载

  • select + proxy : 类加载策略为:lazy=false 同上

  • join + proxy : 使用表链接查询订单以及对应客户信息.lazy属性无效
  • join + false : 使用表链接查询订单以及对应客户信息.lazy属性无效

Hibernate一级缓存

标签 ： Hibernate 一级缓存

缓存

Hibernate中也存在缓存，Hibernate中存在的缓存也是用来提高效率的

在session对象中，有一个缓存

本质：Map 键是ID 值是对象 。hibernate会把所有查询到对象放入缓存中，如果再次查询相同的对象，会从缓存中找，为了提高效率。

Hibernate中存在两种缓存：

  1. 线程级别的缓存：Session缓存
  2. 进程级别的缓存： Hibernate二级缓存

session缓存：就是session对象中存在的缓存，缓存中存在的是（持久化）对象

一级缓存，对象快照

缓存中的快照

在从数据库中取得数据时，会将数据一式两份，一份作为缓存中的对象，一份作为快照（快照与数据库中保持一致），在session提交时作为对比

持久化状态

本质就是存在缓存中的对象，就是持久化状态

session缓存的细节问题

save()方法：
来自于Hibernate

persist() :
来自于JPA接口

HQL语句批量查询

  • HQL语句会使用一级缓存，因为session将结果组装成对象之前，会先检查session中是否已经存在了对象；若已经存在，直接返回已经存在的对象
  • 每次调用HQL查询都会生成sql语句，但是并不代表HQL不会使用一级缓存
  • HQL查询结果会放入缓存当中

SQL语句查询

SQL查询结果会放入一级缓存中，但是前提是要把查询结果封装到对象中。

其他API

  • evict(): 将缓存中的对象移除

  • clear(): 清空一级缓存

  • refresh(): 将缓存中的对象立刻与数据库同步，会再发送一个sql语句

  • saveOrUpdate()
    在代理主键的条件（native）下，saveOrUpdate 可以同时完成保存或更新操作;若主键为空=>save()，若主键有值=> update()

在自然主键条件下（assigned）下，
主键为空=> 报错,因为无论是save还是update 都必须指定id；
主键有值=> 先会根据主键查询数据库。

saveOrUpdate()会先执行select语句查询数据库中的信息。若：
数据库中存在=> 执行update；
数据库中不存在=> 执行insert。

get和load方法的区别：

  • get:get方法被调用时，立刻调用sql语句查询；
  • load:调用时并没有查询数据库，当我们需要使用该对象的时候，才查询数据。

load对象原理和设计：

  • load方法用于提高Hibernate的执行效率（延迟加载）
  • 使用代理技术，创建一个代理对象，属性只有ID有值
  • 返回代理对象
  • 使用代理对象（属性）时，
  • 代理对象调用Hibernate查询数据库，初始化属性

配置文件hibernate.cfg.xml详解

<property name="hbm2ddl.auto">update</property>

/project/etc 中包含诸多配置文件

其中hibernate.properties中可以找到hbm2ddl的配置策

  • update(最常用)

    如果当前数据库中不存在表结构，那么自动创建表结构。如果存在表结构，并且表结构与实体一致，不作修改；若表结构不一致，会修改表结构，但是会保留原有列。

  • creat（极少使用）

    无论是否存在表结构，每次启动Hibernate都会重新创建表结构，即数据会丢失。

  • creat-drop(极少使用)

    无论是否存在表结构，每次启动Hibernate都会重新创建表结构，即数据会丢失。每次Hibernate运行结束时，删除表结构。

  • validate

    不会自动创建表结构，也不会自动维护表结构。Hibernate只校验表结构，如果表结构不一致会抛出异常

Hibernate对象状态

Hibernate对象状态

标签（空格分隔）： Hibernate 对象状态

Hibernate中的对象状态有三种：

瞬时态/临时态

  1. 没有与Hibernate产生关联
  2. 与数据库中的记录没有产生关联

持久态

  1. 与Hibernate有关联
  2. 对象有ID

游离态

  1. 没有与Hibernate产生关联
  2. 对象有ID

当主键自增时：
通过debug发现save()方法仅是获得id,在commit处才会insert

当increment时:
select max(id)

当assigned时：
需要手动指定主键，不指定将会报错

三种状态的作用

持久状态，我恶魔你使用Hibernate主要是为了持久化我们的数据，对于对象的状态，我们期望我们需要同步到数据库的数据，都被转换成持久化状态

持久化状态的特点：Hibernate会自动将持久化状态对象的变化同步到数据库中
